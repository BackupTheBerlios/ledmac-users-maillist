Dear Friends,

as you can have seen, previous days were intensif in ledmac / ledpar activity.

Actually, in CTAN, the published version is 0.18.1 / 0.13. 

Theses versions have some bugs, due to introduction of optional argument in \Xfootnotes. These bug have been corrected this week (and this morning !). The official stable version is 0.18.3/0.13.2. I post it in the CTAN this morning, but it seems CTAN webmasters are not actif yet. I hope they will publish it, but ...

In //, I prepare an ledmac and ledpar 1.0 version.

The main idea of these versions is to provide a new command : \newseries{series}, which add new series of Xfootnote Xend footnoteX etc. For example : \newseries{F,G}. This implies some code refactoring, which is, in all case, needed to help to add new features in future.

One little change is also that the formating options like \nolemmaseparator can take a list of series in first args : 

<code>
\nolemmaseparator[B,C] 
</code>
instead of :
<code>
\nolemmaseparator[B]
\nolemmaseparator[C]
</code>

Actually, I think the code is stable, but the documentation need to be reformating. But as I wanted to have some feedbacks, I propos you to test the β 1.

To do it :
- download https://github.com/maieul/ledmac/zipball/mac1.0%CE%B21-par1.0%CE%B21-arab0.1
- compile the .ins files to obtain the .sty files
- put the .sty files aside your .tex file
- try to compile
- send me a email to explain errors you will see, or better, open a ticket in  https://github.com/maieul/ledmac/issues (it's free)


Note that \addfootins and \addfootinsX are deprecated : and error message will printed, explaining that you should use \newseries instead.
