Dear Friend, 

talking with Dominiki about the break compatibility in Ledmac, I agreed with his idea to provide 2 versions of ledmac/ledpar.

1) Ledmac and Ledpar 0, wich won't be maintened anymore. The version 0.19 will be corresponding with the old version 0.17, because the break compatibility of overload was made on 0.18
2) Version 1.0. The main idea of this version is that people should'nt have to overload any "major" macro of ledmac, because 
	1) They should'nt have to be a hacker.
	2) This technic is dangerous for compatibility descendant, because it implies no change in the API.

But as you know, it's not possible in LaTeX to specify which version of package we Load.

That is why, the version 1 will be distributed by the CTAN with an other name : ledmac1 and ledpar1 (ledarab will be unmaintend too, because I think we should RTL possibility of moderne processeur).

The manual of the version 0.1 will invite to use version 1 instead. For people who didn't hack ledmac (i.e how did'nt made their own footnote series of formating), it will be transparent. For people who hacked, they should make some adaptation, but they will have a more clean code.

I think I will print a error message in the version 0.x which invite to use version 1.x instead, or to add a command to hide this message.

The manual of ledmac 1.0 will suppress all overload description, which can be problematic. I will try to implement some hook instead of these overload. I think I will have only one : the font family for lemma and footnote, to have ≠ font in ≠ series.

I hope to publish one beta 4 this week, to publish official version this week-end. 

I ask for your opinion.
